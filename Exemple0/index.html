Conceptes sinònims Llibreria, API i framework?

La diferència entre els 3 conceptes no és trivial, depèn de l’entorn on estiguem per entendre-ho millor, tot i així aquí hi ha un xic de llum a aquest garbuix de conceptes semblants:

Una library(biblioteca) és una col•lecció de codi construïda per realitzar tasques habituals. El codi d’una library tendeix a ser estable i lliure d’errors. L’ús adequat d’una llibreria pot reduir la quantitat de codi que caldrà escriure.

Una API (Application Programming Interface) és una interfície amb alguna funcionalitat que permet a una aplicació accedir a aquesta funcionalitat.

Les API’s defineixen com utilitzar “alguna cosa més” en termes de: quins objectes i mètodes es poden fer servir com origen de la informació que cal utilitzar. Per exemple: per integrar Facebook en un lloc cal utilitzar la seva API per assegurar que les dades s’actualitzin correctament.

Un framework és una col•lecció d’API’s dissenyada per simplificar la creació d’aplicacions. Les frameworks proporcionen implementacions per components reutilitzables. Els bons framesworks tendeixen a especialitzar-se en les seves aplicacions.
